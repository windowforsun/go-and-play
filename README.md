# 고 앤 플레이 (go-and-play)

사다리 타기·룰렛·손가락 짚기 등 **내기하거나 술래를 정하는 게임 모음**.
브라우저에서 돌아가는 정적 웹앱으로, GitHub Pages에 배포한다.

- 기술 스택: **순수 HTML / CSS / JS (ES 모듈, 빌드 단계 없음)**
- 구조: **허브 + 게임 모듈** (게임을 하나씩 붙여나감)
- 디자인: **다크 젤리 팝 + 마스코트 "말랑이"** (라이트/다크 토글, 기본 다크)
- 온라인(멀티플레이)은 이후 외부 실시간 서비스로 확장 가능 (프론트는 그대로 유지)

> 📄 상세 설계는 [`docs/superpowers/specs/2026-07-19-go-and-play-design.md`](./docs/superpowers/specs/2026-07-19-go-and-play-design.md) 참고 (게임 카탈로그 13종·아키텍처·단계 계획 포함).

## 📁 디렉터리 구조

```
go-and-play/
├── README.md
├── docs/superpowers/specs/   설계 문서
├── brainstorm/               ⚠️ 브레인스토밍 전용 — 정식 코드 아님 (버려질 수 있음)
│   ├── README.md
│   └── concepts/             디자인 컨셉 목업 (A/B/C + C-다크)
└── (정식 앱 코드는 구현 단계에서 추가 — 스펙 §9 구조)
```

> ⚠️ **중요**: `brainstorm/` 아래 파일은 **아이디어 검토용 목업**이며 **정식 코드가 아니다.**
> 정식 앱 코드와 절대 섞지 않는다. 자세한 내용은 [`brainstorm/README.md`](./brainstorm/README.md) 참고.

## 개발 실행

ES 모듈 때문에 `file://` 직접 열기는 차단된다. **로컬 정적 서버**로 실행한다:

```bash
python3 -m http.server 8000   # → http://localhost:8000
```
